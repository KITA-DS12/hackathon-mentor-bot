# 🎯 ユーザー向け使い方ガイド

## 🚀 概要

Hackathon Mentor Botを使って、ハッカソン中の質問や困りごとを効率的に解決する方法を説明します。

## 💡 質問の投稿

### 基本的な質問投稿
```
/mentor-help
```

質問方法を3つから選択できます：

### 🚀 自由投稿（おすすめ）
- **最も簡単**: 質問内容だけ入力すればOK
- **制限なし**: どんな内容でも気軽に投稿
- **迅速**: すぐにメンターに通知

### 📝 シンプル投稿
- **基本項目**: 質問内容 + カテゴリ・緊急度（任意）
- **バランス**: 情報整理しつつ簡潔
- **効率的**: 適度な情報でメンターが理解しやすい

### 📋 詳細投稿
- **完全版**: すべての項目を詳細に記入
- **高品質**: メンターが最も回答しやすい形式
- **推奨**: 複雑な技術的問題の場合

## 📋 質問項目の詳細

### 必須項目
- **質問内容**: あなたの困りごとを具体的に記述

### 選択項目（投稿方法により異なる）
- **カテゴリ**: 
  - 🔧 技術的な問題
  - 🎨 デザイン・UI/UX  
  - 💼 ビジネス・企画
  - 📦 その他

- **緊急度**:
  - 🔴 至急: 作業が完全に止まっている
  - 🟡 普通: 困っているが少し待てる
  - 🟢 低め: 時間があるときに聞きたい

- **相談方法**:
  - 🚀 すぐ相談したい: 即座にメンターに通知
  - ⏰ 予約して相談: 指定時間にメンターに通知

### 任意項目（詳細投稿のみ）
- **現在の状況**: 試したことや調べたこと
- **関連コード・リンク**: GitHub、CodePen、参考サイトのURL
- **エラーメッセージ**: 出ているエラーがあれば貼り付け

## 📤 質問投稿の例

### 良い質問の例
```
【質問内容】
ログイン機能を実装していますが、「Cannot read property 'id' of undefined」
というエラーが出て動作しません。

【現在の状況】
- React + Node.jsで開発
- 公式ドキュメントを確認済み
- console.logでデバッグを試した

【関連コード】
https://github.com/user/project/blob/main/login.js

【エラーメッセージ】
TypeError: Cannot read property 'id' of undefined
    at login.js:45:12
```

### 簡潔でも十分な例
```
React のuseEffectが無限ループしてしまいます。
依存配列を空にしても解決しません。助けてください！
```

## ⏰ 予約相談機能

### 予約時間の選択
- **30分後**: 少し調べてから相談したい
- **1時間後**: 他の作業を完了してから
- **2時間後**: まとまった時間を確保してから
- **明日午前**: 今日は時間がない場合

### 自動解決確認
- 予約時間の5分前に自動確認メッセージ
- 「解決済み」を選択すると予約キャンセル
- 「まだ未解決」でメンターに通知を送信

## 🔔 メンター通知とマッチング

### 通知の仕組み
1. 質問を投稿すると対応可能なメンター（最大5名）に自動通知
2. カテゴリに関係なく全メンターに平等に通知
3. メンターが「対応開始」ボタンで対応開始

### 通知例
```
🔔 技術的な問題 の質問です
@mentor1 @mentor2 @mentor3

【質問内容】
ReactでAPIからデータを取得できません
...

[対応開始] [詳細確認]
```

## 💬 メンターとのやり取り

### スレッドでの対話
1. メンターが「対応開始」を押すと専用スレッドが作成
2. 質問者とメンターが自動的にスレッドに招待
3. スレッド内で詳細なやり取りを実施

### 効果的なコミュニケーション
- **具体的に**: 「動かない」ではなく「〜のエラーが出る」
- **情報提供**: 環境、言語、フレームワークを明記
- **試したこと**: 既に試した解決方法を共有
- **感謝の気持ち**: メンターはボランティアです

## 📊 対応状況の確認

### ステータス表示
質問の状況がリアルタイムで表示されます：

- 🟡 **対応待ち**: メンターの対応待ち
- 🔵 **対応中**: メンターが対応中
- 🟠 **中断中**: メンターが一時中断
- ✅ **完了**: 問題解決済み

### 対応完了の確認
- メンターが「完了」をマークすると通知
- 解決したかどうかの確認メッセージ
- 追加質問があれば遠慮なく投稿

## 🔄 フォローアップ機能

### 自動フォローアップ
- 質問投稿から **30分後** に自動確認
- 解決していない場合は **2時間後** に再確認
- 「未解決」を選択するとメンターに再通知

### フォローアップへの対応
```
質問投稿から30分が経過しました。
問題は解決しましたか？

[✅ 解決済み] [❌ まだ未解決]
```

- **解決済み**: フォローアップ終了
- **まだ未解決**: メンターに再通知、優先対応

## 🎯 効果的な質問のコツ

### 質問前の準備
1. **エラーメッセージをコピー**: 正確な情報が重要
2. **環境情報を整理**: OS、ブラウザ、言語バージョン
3. **試したことをメモ**: 重複を避けるため
4. **コードを準備**: 問題の箇所を特定

### 質問文の書き方
- **結論から**: 「〜ができません」を最初に
- **具体的に**: 「動かない」より「〜のエラーが出る」
- **簡潔に**: だらだらと長文にしない
- **情報十分**: 必要な情報は省略しない

### やり取りのマナー
- **返信は迅速に**: メンターが待っている場合は早めに
- **感謝を表す**: 「ありがとうございます」の一言
- **結果を報告**: 「解決しました」の報告
- **学びを共有**: 他の人の参考になるかも

## 🚫 こんな時はどうする？

### よくある状況と対処法

**Q: メンターから返信がない**
- 少し待ってから再度質問投稿
- 緊急度を「至急」に変更
- 別の時間帯に再投稿

**Q: 解決方法が理解できない**
- 「もう一度詳しく教えてください」と遠慮なく質問
- 「〜の部分がわかりません」と具体的に
- 画面共有やコードレビューをお願い

**Q: 複数の問題がある**
- 一つずつ別々に質問投稿
- 関連がある場合は「先ほどの質問の続きです」と明記

**Q: 緊急で困っている**
- 緊急度を「🔴至急」に設定
- 「すぐ相談したい」を選択
- 質問内容に「緊急」と明記

## 📞 困った時のサポート

### トラブルシューティング
- **ボットが反応しない**: 管理者に連絡
- **モーダルが表示されない**: ページを再読み込み
- **通知が来ない**: Slack通知設定を確認

### 改善要望・フィードバック
- 使いにくい機能があれば管理者まで
- 新機能のアイデアも歓迎
- バグや不具合の報告
