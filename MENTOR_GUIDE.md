# メンターガイド（マルチチャンネル対応）

Hackathon Mentor Botを使って効果的に参加者をサポートする方法を説明します。

## 🚀 メンター登録

### 登録
```
/mentor-register
```
- **名前**: 参加者に表示される名前
- **自己紹介**: 専門分野・経験・サポート領域
- **初期ステータス**: available/busy/offline

### 登録解除
```
/mentor-unregister
```

## 📋 メンター向けコマンド

| コマンド | 機能 |
|---------|------|
| `/mentor-register` | メンター登録 |
| `/mentor-unregister` | 登録解除 |
| `/mentor-list` | メンター一覧表示 |
| `/mentor-questions` | **質問一覧・管理（重要）** |
| `/mentor-status` | ステータス変更 |
| `/mentor-health` | システム状態確認 |

## 📊 質問管理（重要）

### `/mentor-questions` で確認
- **📊 チャンネル別統計**: 複数チャンネルの質問分布を表示
- **🚨 要注意質問**: 担当者不在・24時間以上経過
- **🟡 待機中**: 対応待ちの新しい質問
- **🟠 中断中**: 一時中断された質問
- **🔵 対応中**: 現在対応中の質問

### 定期チェック推奨
- **朝**: 要注意質問をチェック
- **昼**: 新しい待機中質問を確認
- **夕**: 長期未完了質問の確認

## 📈 ステータス管理

### `/mentor-status` で変更
- **🟢 available**: 対応可能（積極的に対応）
- **🟡 busy**: 忙しい（緊急時のみ）
- **🔴 offline**: 対応不可（メンション対象外）

## 🔔 質問対応の流れ（マルチチャンネル対応）

### 1. 通知受信
- **メンターチャンネル**で質問通知を受信
- 通知には質問内容とチーム情報が含まれます

### 2. 対応開始
「対応開始」ボタンをクリック：
- 質問が「対応中」に変更
- **質問元のチームチャンネル**で自動スレッド作成
- 質問者とメンターがスレッドに招待

### 3. チームチャンネルでの対応
- **質問投稿されたチームチャンネル**のスレッド内で対応
- 「中断」「完了」ボタンで状態管理
- チームメンバーも会話を確認可能

### 4. 対応完了
「完了」ボタンまたは質問者の「✅ 解決済み」ボタンで：
- 質問が「完了」状態に変更
- 質問者にDM通知

## 🏢 マルチチャンネル対応の特徴

### 通知システム
- **一元通知**: 全チャンネルの質問がメンターチャンネルに通知
- **詳細情報**: 質問内容、チーム名、チャンネル情報を表示
- **直接リンク**: 「質問を見る」ボタンで元の質問に直接ジャンプ

### 対応場所
- **スレッド場所**: 質問投稿されたチームチャンネル内
- **チーム透明性**: チームメンバー全員が対応過程を確認可能
- **コンテキスト保持**: チーム固有の技術スタックや課題に対応

### メンター管理
- **統合管理**: `/mentor-questions`で全チャンネルの質問を一覧表示
- **チャンネル統計**: 複数チャンネル間の質問分布と状況を把握
- **効率的対応**: メンターチャンネルから全体状況を把握
- **適切な引き継ぎ**: チャンネル情報を含めた詳細な引き継ぎ

## 🔄 高度な管理機能

### 中断・再開
- **中断**: 一時離席時に使用
- **再開**: 元担当者のみ可能
- **引き継ぎ**: 他メンターが引き継ぎ可能

### 担当交代
- **担当解除**: 他メンターに引き継ぎ
- 解除後は「待機中」に戻る
- チーム情報は保持される

### チーム対応
- 複雑問題は複数メンターで対応
- チームチャンネルのスレッドに他メンターを招待
- チーム固有の知識を共有

## 💡 効果的なメンタリング

### 初期対応
1. **素早い反応**: 30分以内を目標
2. **チーム理解**: チーム構成と技術スタックを確認
3. **情報収集**: 環境・エラー・試行内容を確認
4. **方向性提示**: チームの文脈に合わせた解決アプローチ

### コミュニケーション
- **チーム意識**: チームメンバー全員が学べるような説明
- **具体的な指示**: チームの技術レベルに合わせた指示
- **励ましの言葉**: チーム全体のモチベーション向上
- **学習機会**: チーム全体のスキルアップに貢献

## 🔧 トラブルシューティング

### よくある問題
- **対応開始ボタンが押せない**: 他メンターが既に対応中
- **スレッド作成されない**: 元チャンネルの権限を確認
- **質問内容不明**: 「詳細確認」ボタンで追加情報確認
- **チャンネルアクセス不可**: 管理者にチャンネル招待を依頼

### マルチチャンネル特有の注意点
- **チャンネル権限**: 対応するチームチャンネルへの参加が必要
- **通知場所**: 質問通知はメンターチャンネル、対応はチームチャンネル
- **情報の一貫性**: チャンネル間で情報が分散しないよう注意

## ⚡ ベストプラクティス

### DO（推奨）
- ✅ 素早い初期対応（30分以内）
- ✅ チームコンテキストを理解した説明
- ✅ チーム全体の学習を意識したアドバイス
- ✅ 適切な引き継ぎ・連携（チャンネル情報含む）
- ✅ 定期的な質問一覧チェック（全チャンネル）

### DON'T（避ける）
- ❌ チームのコンテキストを無視した対応
- ❌ 一方的な答えの押し付け
- ❌ チャンネル間での情報の重複投稿
- ❌ 中途半端な対応での放置
