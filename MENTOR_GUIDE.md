# 📚 メンター向け使い方ガイド

## 🎯 概要

Hackathon Mentor Botを使って、ハッカソン参加者からの質問に効率的に対応する方法を説明します。

## 🚀 メンター登録

### 1. メンター登録
```
/mentor-register
```
- **名前**: 表示名を入力
- **自己紹介**: 経歴や得意分野を簡潔に記入
- **ステータス**: 対応可能 / 忙しい / 対応不可 から選択

### 2. 登録確認
```
/mentor-list
```
- 現在登録されているメンター一覧を確認（あなたにのみ表示）
- 各メンターのステータス（🟢対応可能 / 🟡忙しい / 🔴対応不可）を確認

## 📊 ステータス管理

### ステータス変更
```
/mentor-status
```
- あなたの現在のステータスを変更
- **対応可能**: 質問対応が可能
- **忙しい**: 緊急時のみ対応
- **対応不可**: 質問対応ができない状態

💡 **ベストプラクティス**: 作業状況に応じてこまめにステータスを更新しましょう

## 🔔 質問通知の受信

### 質問投稿時の通知
- 質問が投稿されると、利用可能なメンター全員にメンション通知
- 最大5人のメンターに同時通知
- カテゴリに関係なく、全メンターに平等に通知

### 通知の例
```
🔔 技術的な問題 の質問です
@mentor1 @mentor2 @mentor3

【質問内容】
ログイン機能でエラーが出て困っています
...
```

## 💬 質問対応フロー

### 1. 質問への対応開始
- 質問メッセージの **「対応開始」** ボタンをクリック
- 自動的に専用スレッドが作成される
- 質問者とメンターがスレッドに招待される

### 2. スレッドでの対応
- 作成されたスレッド内で質問者と直接やり取り
- 必要に応じて画面共有やコードレビューを実施
- 解決まで継続的にサポート

### 3. 対応状況の管理
質問の状況に応じてボタンで管理：

- **🔵 対応中**: 現在対応している状態
- **🟠 中断**: 一時的に対応を中断
- **✅ 完了**: 質問が解決して対応完了

### 4. 対応完了
- **「完了」** ボタンで対応終了をマーク
- 質問者に解決確認の通知が送信される

## 🔄 フォローアップ対応

### 自動フォローアップ通知
- 質問投稿から30分後・2時間後に自動確認
- 未解決の場合、メンターに再通知
- 質問者が「未解決」を選択した場合に通知を受信

### フォローアップ対応
1. フォローアップ通知を受信
2. まだ対応可能であれば「対応開始」
3. 引き続きスレッドで質問者をサポート

## 📋 質問の詳細確認

### 詳細確認ボタン
- **「詳細確認」** ボタンで質問の全情報を表示
- 質問内容、カテゴリ、緊急度、関連情報などを確認
- 対応判断の参考に活用

### 表示される情報
- 📝 質問内容
- 🏷️ カテゴリ（技術的な問題、デザイン・UI/UX等）
- ⚡ 緊急度（🔴至急 / 🟡普通 / 🟢低め）
- 📍 現在の状況
- 🔗 関連リンク・コード
- ❌ エラーメッセージ

## 🎯 効果的な対応のコツ

### 質問対応のベストプラクティス

1. **迅速な初動対応**
   - 質問を見たら可能な限り早く「対応開始」
   - 忙しい場合は一言「少々お待ちください」と返信

2. **具体的な質問**
   - 「エラーの詳細な内容は？」
   - 「どの部分のコードでエラーが発生しますか？」
   - 「試したことがあれば教えてください」

3. **段階的な解決**
   - 問題を小さく分割して一つずつ解決
   - 解決手順を明確に説明
   - 質問者が理解できるレベルで説明

4. **リソースの提供**
   - 参考になるドキュメントのリンク
   - サンプルコードの提供
   - 類似問題の解決例

5. **確実な完了確認**
   - 解決したかを必ず確認
   - 追加の質問がないかチェック
   - 「完了」ボタンで正式に終了

## 🗑️ メンター登録解除

### 登録解除の方法
```
/mentor-unregister
```
- 確認画面が表示される
- **「🗑️ 解除する」** で登録解除
- **「❌ キャンセル」** で取り消し

### 解除後の影響
- 今後の質問でメンション通知を受信しなくなる
- 過去の対応履歴は保持される
- 再登録は `/mentor-register` でいつでも可能

## 📞 サポート・質問

### トラブル時の対応
- ボットが応答しない場合: 管理者に連絡
- 通知が来ない場合: ステータスが「対応不可」になっていないか確認
- スレッドが作成されない場合: 再度「対応開始」ボタンを押す

### 改善提案
- 機能追加や改善のアイデアがあれば管理者まで
- 使いにくい点があれば遠慮なくフィードバック

- **完了**: 対応完了をマーク
