# メンターガイド

このガイドでは、Hackathon Mentor Botを使ってメンターとして効果的に参加者をサポートする方法を説明します。

## 🚀 メンター登録

### 1. メンター登録

```
/mentor-register
```

登録フォームで以下を入力：
- **名前**: 参加者に表示される名前
- **自己紹介**: 専門分野・経験・サポート可能な領域
- **初期ステータス**: available/busy/offline から選択

### 2. 登録完了
- 登録後、質問投稿時に自動でメンションが送信されます
- いつでも `/mentor-unregister` で登録解除可能


## 📋 メンター向けコマンド一覧

### 基本コマンド
```
/mentor-register     # メンター登録
/mentor-unregister   # メンター登録解除
/mentor-list         # 登録メンター一覧表示
/mentor-questions    # 質問一覧・管理（重要！）
/mentor-status       # ステータス確認・変更
```

## 📊 質問管理機能（重要）

### `/mentor-questions` コマンド

このコマンドで全質問の状況を包括的に確認できます：

#### 🚨 要注意質問（最優先表示）
- **担当者不在**: 登録されていないメンターIDが担当の質問
- **24時間以上経過**: 長期間放置されている質問
- 問題の種類を明示して警告表示

#### 🟡 待機中の質問
- メンターの対応を待っている状態
- 「対応開始」ボタンで対応可能
- 詳細確認ボタンで内容を事前確認

#### 🟠 中断中の質問
- 他のメンターが一時中断した質問
- 「対応再開」ボタンで再開（元担当者のみ）
- 「担当解除」ボタンで他のメンターに引き継ぎ

#### 🔵 対応中の質問
- 現在進行中の対応状況
- 担当メンター情報を表示
- 「担当解除」で他のメンターに引き継ぎ可能

### 定期チェックの推奨
- **朝**: `/mentor-questions` で要注意質問をチェック
- **昼**: 新しい待機中質問を確認
- **夕**: 長期未完了質問の状況確認

## 📈 ステータス管理

### `/mentor-status` コマンド

3つのステータスから選択：
- **🟢 available**: 対応可能（積極的に質問対応）
- **🟡 busy**: 忙しい（緊急時のみ対応）
- **🔴 offline**: 対応不可（メンション送信対象外）

### ステータスの活用
- **ハッカソン開始**: available に設定
- **集中作業時**: busy に設定
- **休憩・食事**: offline に設定
- **ハッカソン終了**: offline に設定

## 🔔 質問対応の流れ

### 1. 質問通知の受信
- 質問投稿時に**全メンター**に自動メンション
- 通知には質問内容とボタンが表示

### 2. 対応開始
「対応開始」ボタンをクリックすると：
- 質問が「対応中」状態に変更
- 自動でスレッドが作成
- 質問者とメンターがスレッドに招待
- 質問者にDMで対応開始通知

### 3. スレッドでの対応
- スレッド内で質問者と直接やり取り
- 「中断」「完了」ボタンで状態管理
- 必要に応じて他のメンターを @mention で招待

### 4. 対応完了
「完了」ボタンをクリックすると：
- 質問が「完了」状態に変更
- 質問者にDMで完了通知
- フォローアップが自動停止

## 🔄 高度な質問管理

### 中断・再開機能
- **中断**: 一時的に離席する場合に使用
- **再開**: 中断した質問を後で再開
- 元担当者のみ再開可能

### 担当者交代機能
- **担当解除**: 対応困難な場合に他のメンターに引き継ぎ
- 解除すると質問が「待機中」に戻る
- 他のメンターが「対応開始」で引き継ぎ可能

### チーム対応
- 複雑な問題は複数メンターでの対応も可能
- スレッドに他のメンターを招待
- 専門分野に応じた連携

## 💡 効果的なメンタリングのコツ

### 初期対応
1. **素早い反応**: 質問から30分以内の対応を目標
2. **状況確認**: 「どこまで進んでいますか？」
3. **情報収集**: 環境、エラー内容、試したことを確認
4. **方向性提示**: 複数の解決アプローチを提案

### 問題解決アプローチ
1. **問題の特定**: 何が根本原因か
2. **優先順位**: 最も効果的な解決策から
3. **段階的解決**: 小さなステップに分割
4. **検証**: 各ステップで動作確認

### コミュニケーション
- **丁寧な言葉遣い**: 初心者にも分かりやすく
- **具体的な指示**: 「〜を試してください」
- **励ましの言葉**: 「いい質問ですね」「頑張っていますね」
- **学習機会**: なぜその解決策なのかも説明

## 📊 質問監視・管理

### 定期確認項目
- [ ] 新しい待機中質問はないか
- [ ] 長期間放置されている質問はないか
- [ ] 担当者不在の質問はないか
- [ ] 中断中で再開待ちの質問はないか

### 問題のある質問への対応
1. **担当者不在**: 代わりに対応開始
2. **長期未完了**: 状況確認・引き継ぎ検討
3. **中断放置**: 他のメンターが引き継ぎ
4. **複雑すぎる**: 専門メンターにエスカレーション

## 🔧 トラブルシューティング

### よくある問題

**Q: 対応開始ボタンが押せない**
A: 他のメンターが既に対応開始している可能性があります

**Q: スレッドが作成されない**
A: ブラウザを更新するか、管理者に連絡してください

**Q: 質問内容が理解できない**
A: 「詳細確認」ボタンで追加情報を確認してください

**Q: 解決できない問題**
A: 「担当解除」で他のメンターに引き継いでください

## ⚡ メンタリングベストプラクティス

### DO（推奨）
- ✅ 素早い初期対応（30分以内）
- ✅ 丁寧で分かりやすい説明
- ✅ 学習につながるアドバイス
- ✅ 適切な引き継ぎ・連携
- ✅ 定期的な質問一覧チェック

### DON'T（避ける）
- ❌ 長時間の放置
- ❌ 一方的な答えの押し付け
- ❌ 専門用語だけの説明
- ❌ 中途半端な対応での放置
- ❌ 質問の批判・否定
